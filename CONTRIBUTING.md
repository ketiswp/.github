# 🤝 ketiswp 협업 가이드

## 🌿 브랜치 규칙

- `main`은 항상 동작 상태 유지. 직접 push 금지.
- 작업은 브랜치 만들어서:
  - `feature/` — 새 기능, 개선
  - `fix/` — 버그 수정

## ✍️ 커밋 메시지

- 첫 줄은 30자 이내 한 줄 요약
- 가능하면 접두사 사용:
  - `feat:` 새 기능
  - `fix:` 버그 수정
  - `docs:` 문서

## 🔄 PR 워크플로우

브랜치 작업 → push → PR 생성 → 리뷰어로 @devcow85 지정 → 리뷰 후 Squash and merge → 브랜치 삭제
